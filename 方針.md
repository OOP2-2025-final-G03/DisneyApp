# 以下のデータを登録、編集が行える
- ユーザー（User）(k24019)
  - ID、名前、年齢、性別、身長
- エリア(Area) (k24100)
  - ID、名前、アトラクションのID
- アトラクション(Attraction) (k24039)
  - ID、名前、参加条件（身長、年齢制限など）
- アトラクションの参加履歴(history) (k24056)
  - ユーザーID、アトラクションID、参加時間

## 追加/変更したいもの
- ユーザー
  - ユーザを追加するときにIDは自動的に割り当てられたい（ランダムまたは連番（`0001`,`0002`,`0003`...））、追加するときに同じIDがすでにあれば、新しいIDを割り当てるようにしたいと思います。
  >*現在はユーザーIDが重複した場合、そのままpeeweeのエラーが出している*

- エリア、アトラクション
  - IDがない、追加してください。
  >もし追加するときに同じIDがすでにあれば、簡単に割り当てられただけでいいです(id+1などでいいが, 重複しないようにしてください)。

- エリア
  - 複数のアトラクションを持つことができるようにしたいと思います。
>実現方法(仮定)
><br>エリアのIDを設定する、アトラクションは新しくareaフィールドを作ってその中にエリアを格納する。
><br>Area一覧画面で、各エリアに紐づくAttractionを表示したい場合は、area.attractions で取得できます.

- アトラクション
  - Attractionの追加・編集画面で、エリア（Area）を選択するプルダウンを設置してください。
> プルダウンの中身は、Areaモデルのnameを使ってください。




## 簡単な資料
### フィールド型一覧（SQLite）
| フィールド型 | SQLite型 | 説明 |
|-------------|----------|----------------|
| AutoField | integer | 自動的に値が増加する整数型（主キー向け） |
| BigAutoField | integer | 大きな範囲を持つ自動増分整数型 |
| IntegerField | integer | 通常の整数型 |
| BigIntegerField | integer | 64ビット相当の大きな整数を扱う |
| SmallIntegerField | integer | 小さな範囲の整数型 |
| IdentityField | 未対応 | SQLiteでは IDENTITY 列はサポートされていない |
| FloatField | real | 単精度の浮動小数点数 |
| DoubleField | real | 倍精度の浮動小数点数 |
| DecimalField | decimal | 精度を保った固定小数点数（誤差を避けたい場合に使用） |
| CharField | varchar | 可変長の文字列 |
| FixedCharField | char | 固定長の文字列 |
| TextField | text | 長い文字列（制限なし） |
| BlobField | blob | バイナリデータ（画像・ファイルなど） |
| BitField | integer | ビット列を整数として扱う |
| BigBitField | blob | 大きなビット列をバイナリとして保存 |
| UUIDField | text | UUIDを文字列として保存 |
| BinaryUUIDField | blob | UUIDをバイナリ形式で保存 |
| DateTimeField | datetime | 日付と時刻を保持 |
| DateField | date | 日付のみを保持 |
| TimeField | time | 時刻のみを保持 |
| TimestampField | integer | UNIXタイムスタンプ（秒数） |
| IPField | integer | IPアドレスを数値として保存 |
| BooleanField | integer | 真偽値（0=false, 1=true） |
| BareField | untyped | 型指定を行わない特殊なフィールド |
| ForeignKeyField | integer | 外部キー（他テーブルの主キー参照） |
